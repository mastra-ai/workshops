processors/topic-guard.ts processInput + typed metadata 
type TopicGuardMetadata = {
  category: string; confidence: number;
};

export class TopicGuard
  implements Processor<'topic-guard', TopicGuardMetadata> {
  readonly id = 'topic-guard';
  private classifierAgent = new Agent({
    model: 'openrouter/openai/gpt-oss-safeguard-20b',
    // ↑ fast safeguard model, not the main agent model
  });

  async processInput({ messages, abort }) {
    const { object } = await this.classifierAgent.generate(text, {
      structuredOutput: z.object({ onTopic: z.boolean(), ... }),
    });
    if (!object.onTopic) {
      abort(`Off-topic: ${object.category}`, {
        metadata: { category: object.category, confidence: object.confidence },
      });
    }
    return messages;
  }
}
processors/pii-guard.ts regex only — no LLM! 
export class PIIGuard
  implements Processor<'pii-guard', PIIGuardMetadata> {
  async processInput({ messages, abort }) {
    // Pure regex — runs in microseconds, costs $0
    for (const [type, pattern] of PII_PATTERNS) {
      if (pattern.test(text)) detectedTypes.push(type);
    }
    if (detectedTypes.length > 0)
      abort('PII detected', {
        metadata: { detectedTypes, severity: 'critical' },
      });
  }
}

Typed Metadata
Processor<TId, TMetadata> — the second generic types what abort() metadata contains. Structured data about what triggered the tripwire.
No LLM Needed
PII Guard uses pure regex — runs in microseconds, costs nothing. Not every guard needs an LLM call.
Parallel Workflow
.parallel() runs all three checks concurrently. If any calls abort(), the pipeline stops with that processor's typed metadata.
Safeguard Model
Uses gpt-oss-safeguard-20b for classification — fast, cheap, purpose-built for safety checks.

processors/task-drift-monitor.ts processOutputStep 
async processOutputStep({ text, stepNumber, state, abort }) {
  if (stepNumber < 2 || stepNumber % 2 !== 0) return;
  const { object } = await evaluator.generate(..., {
    structuredOutput: z.object({ alignmentScore: z.number() }),
  });
  if (object.alignmentScore < 0.5)
    abort('Drifted from task', {
      retry: true,
      metadata: { driftScore: object.alignmentScore },
    });
}
processors/cost-tracker.ts processOutputStream 
async processOutputStream({ part, state, writer, abort }) {
  if (part.type === 'text-delta') {
    state.tokens += Math.ceil(part.payload.text.length / 4);
    await writer.custom({
      type: 'data-cost-update',
      data: { tokens: state.tokens, cost: ... },
    });
    if (state.tokens > MAX_TOKENS) abort('Budget exceeded');
  }
  return part;
}
processInputStep
Runs before every LLM call. Can swap model, filter activeTools, modify systemMessages. The most powerful per-step hook.
processOutputStep
After each LLM response, before tool execution. Gets stepNumber, text, toolCalls. Perfect for periodic quality checks with abort({ retry: true }).
processOutputStream
Real-time chunk processing. writer.custom() emits data-* events to the client. Can abort() mid-stream if budget/safety is violated.
state
Mutable object that persists across all method calls within a single request. Track original intent, token counts, flags.

processors/wrap-up-enforcer.ts shared constant 
import { MAX_STEPS } from '../constants';

async processInputStep({ stepNumber, systemMessages }) {
  if (stepNumber < MAX_STEPS - 2) return;
  return {
    systemMessages: [...systemMessages, {
      role: 'system',
      content: `Wrap up. ${MAX_STEPS - stepNumber} steps left.`,
    }],
  };
}

export const enterpriseAgent = new Agent({
  model: 'openai/gpt-5.2',
  maxSteps: MAX_STEPS,
  inputProcessors: [
    inputPipeline,     // regex → parallel LLM
    new ModelRouter(),  // step 0: 5.2, 1+: nano
    new ToolDependencyEnforcer(),
    new WrapUpEnforcer(),
  ],
  outputProcessors: [
    outputPipeline,           // drift every 5th step
    new OrderConfirmation(), // fire webhook on order
    new EscalationDetector(),// flag for human review
  ],
  maxProcessorRetries: 3,
});
Layered Defense
Regex pre-filter runs first (free). Only if it passes do the expensive LLM checks run — and those run in parallel.
Conditional Branching
.branch() only runs drift monitoring every 5th step. Don't pay for checks you don't need every time.
Wrap-Up Enforcer
Uses the same MAX_STEPS constant as the agent. Prevents getting cut off mid-task by injecting wrap-up instructions.
onFinish Side Effects
processOutputResult as an onFinish hook — fire webhooks on orders, detect escalations with a nano model. Business logic, not observability.

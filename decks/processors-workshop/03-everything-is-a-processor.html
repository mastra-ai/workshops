The Insight

Everything is a Processor

Processors are middleware for the agentic loop — hooks that run at specific phases of every LLM interaction.

📥 Input processInput
→
🔄 Each Step processInputStep
→
🤖 LLM generate()
→
✅ Each Step processOutputStep
→
📤 Result processOutputResult

Code Walkthrough

Memory is processors

When you configure memory on an agent, Mastra doesn't do anything magical — it creates processors.

packages/core/src/memory/memory.ts getInputProcessors() 
async getInputProcessors() {
  const processors = [];

  if (config.workingMemory) {
    processors.push(
      new WorkingMemory({ storage, template, scope })
    );
  }

  if (config.lastMessages) {
    processors.push(
      new MessageHistory({ storage, lastMessages })
    );
  }

  if (config.semanticRecall) {
    processors.push(
      new SemanticRecall({ storage, vector, embedder })
    );
  }

  if (config.observationalMemory) {
    processors.push(
      new ObservationalMemory({ storage })
    );
  }

  return processors;
}
Insight
The Memory class returns an array of processors. Each memory feature is just a processor that hooks into processInput and processOutputResult.

This is the same interface you implement when writing custom processors.
WorkingMemory
processInput → injects working memory as system message
MessageHistory
processInput → loads thread history  ·  processOutputResult → persists messages
SemanticRecall
processInput → vector search for context  ·  processOutputResult → embeds messages
ObservationalMemory
processOutputResult → compresses & restructures long conversations

It keeps going

Features you already use — all processors

Skills, tool search, structured output, even prepareStep — when you configure these on an agent, Mastra creates processors behind the scenes.

if (options.prepareStep) {
  inputStepProcessors.push(

SkillsProcessor
processInputStep → discovers skills, injects instructions as system messages, gives the agent an activate-skill tool.
StructuredOutputProcessor
processOutputStream → when you set a model on structuredOutput, it creates an internal agent that transforms unstructured text into validated JSON — streamed as partial objects in real time.
ToolSearchProcessor
processInputStep → adds search_tools and load_tool meta-tools so the agent discovers and loads tools on demand. Dramatically reduces context usage.
PrepareStepProcessor
processInputStep → wraps the legacy prepareStep callback into a processor. Even old APIs become processors internally.
The Pattern
Configure a feature → Mastra creates a processor. Same interface you implement yourself. No magic, just processors.

The Interface

One interface, five hooks

Processor Interface 
interface Processor {
  readonly id: string;

  processInput?()        // once, before agentic loop
  processInputStep?()    // each step, before LLM
  processOutputStream?() // each streaming chunk
  processOutputStep?()   // each step, after LLM
  processOutputResult?() // once, after completion
}

Implement only the hooks you need. Mastra runs them at the right time, in the right order.

Streaming & Multi-Model

One harness, many models

Every component emits events. Streaming is not optional. And the harness is interrupted at any time — design for resumability by default.

Key Design Point
Interrupts happen. The agent might be 500 tokens into a response when the user steers. The harness must handle abort gracefully — save what was streamed, clean up, and restart with the new context.

Model Strategy

Mix models deliberately

Different models for different jobs. The harness routes per mode, supports both API keys and OAuth, and tracks cost across all providers.

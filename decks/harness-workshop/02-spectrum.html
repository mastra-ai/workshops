The Big Picture

Framework vs Harness

Agent frameworks and agent harnesses sit at different points on a spectrum of opinionation. Understanding where they sit matters because it changes what you're responsible for.

Raw Code
API Calls
Call APIs directly. Manage state yourself. Total flexibility, total responsibility.
Framework
Building Blocks
Structure & abstractions. You pick memory, configure tools, define orchestration. Modular & swappable.
Harness
Complete System
Everything baked in. Memory, context management, the agent loop, safety. All decided for you.

The Distinction

What's actually different?

A framework gives you abstractions for building agents. A harness gives you a complete system.

Framework

For Builders

You define roles, tasks, tools. You specify how agents coordinate. The framework handles the plumbing — but you're making the architectural decisions.

  • Opinionated about building blocks
  • Memory, tools, LLM providers are swappable
  • You compose the system
  • You understand how pieces fit together
Harness

For Users

You add your API keys, point it at tools, and it runs. Memory, context management, the agent loop, safety checks — all decided for you.

  • Opinionated about everything
  • Decisions are baked in
  • You point it at a task
  • Trade control for speed

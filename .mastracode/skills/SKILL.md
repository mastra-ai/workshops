# Skill: Slide Deck Presentation Builder

Build interactive, dark-themed HTML slide decks using the partial-based build system in this project. Decks live in `decks/[deck-name]/`, share a global design system from `shared/`, and are built to `dist/` via `node build.js`.

---

## Repository Structure

```
presentations/
├── build.js                      # Build script (partial injection + topnav generation)
├── package.json                  # Scripts: build, dev, new-deck
├── shared/                       # Global design system
│   ├── shared.css                # Variables, components, utilities
│   ├── shared.js                 # Navigation, scroll effects, animation helpers
│   └── partials/                 # Reusable HTML fragments
│       ├── head.html             # <meta>, fonts, shared.css, <title> placeholder
│       ├── topnav.html           # (template — generated by build.js)
│       └── scripts.html          # shared.js include
├── decks/
│   ├── om-workshop/              # Example deck
│   │   ├── deck.json             # Deck metadata
│   │   ├── index.html            # Redirect to first slide

│   │   └── ...
│   └── component-showcase/       # Component reference deck
│       ├── deck.json
│       ├── index.html
│       ├── 01-overview.html
│       └── ...
├── examples/                     # Workshop code examples
└── dist/                         # Build output (gitignored)
```

## Creating a New Deck

### 1. Create the deck directory

```bash
mkdir decks/my-deck
```

### 2. Create `deck.json`

```json
{
  "title": "My Deck Title",
  "slides": [
    { "file": "01-intro.html", "nav": "Intro" },
    { "file": "02-topic.html", "nav": "Topic" },
    { "file": "03-conclusion.html", "nav": "Conclusion" }
  ]
}
```

- `title` — displayed in the topnav brand and page titles.
- `slides` — ordered list. `file` is the filename, `nav` is the label shown in the topnav.
- Slides must be listed in navigation order. The build system generates prev/next links automatically.

### 3. Create `index.html`

```html
<!DOCTYPE html>
<html lang="en">
<head><meta http-equiv="refresh" content="0; url=01-intro.html"></head>
<body></body>
</html>
```

### 4. Create slide files

Use partial markers instead of hardcoded boilerplate:

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <!-- partial:head -->
    <style>
        /* Per-slide custom styles */
    </style>
</head>
<body>

<!-- partial:topnav -->

<div class="page">
    <main class="main">
        <!-- Slide content here -->
    </main>
</div>

<!-- partial:scripts -->
<!-- partial:keynav -->
</body>
</html>
```

### 5. Build

```bash
npm run build     # or: node build.js
```

Output goes to `dist/[deck-name]/`. Each deck gets its own directory with all slides, `shared.css`, and `shared.js`.

## Partial Markers

| Marker | Replaced With |
|---|---|
| `<!-- partial:head -->` | Meta tags, font links, `<title>`, shared.css link |
| `<!-- partial:topnav -->` | Auto-generated topnav with all slide links for this deck |
| `<!-- partial:scripts -->` | `<script src="shared.js"></script>` |
| `<!-- partial:keynav -->` | `<script>initKeyNav(prevUrl, nextUrl);</script>` (auto-calculated) |

The `<!-- partial:title -->` inside `head.html` is replaced with `"[Nav Label] — [Deck Title]"`.

**Important:** Place `<!-- partial:scripts -->` and `<!-- partial:keynav -->` just before `</body>`. If you have inline `<script>` blocks, place them between `<!-- partial:scripts -->` and `<!-- partial:keynav -->`.

## Layout Patterns

### Full-Page Centered (Hero)

For title slides and single-focus content:

```html
<div class="page">
    <main class="main">
        <div class="hero">
            <h1>Big Title</h1>
            <p class="subtitle">Supporting text</p>
        </div>
    </main>
</div>
```

```css
.page { display: block; }
.main { margin-left: 0; max-width: 100%; padding: 0 4rem; }
.hero {
    display: flex;
    flex-direction: column;
    align-items: center;
    justify-content: center;
    text-align: center;
    min-height: calc(100vh - var(--topnav-height));
    padding: 2rem;
    gap: 2rem;
}
```

### Scrollable Sections

For multi-section content slides:

```html
<div class="page">
    <main class="main">
        <section class="section" id="topic-one">
            <span class="section-label">01 — Topic</span>
            <h2>Section Title</h2>
            <p>Content...</p>
        </section>
        <section class="section" id="topic-two">
            <!-- ... -->
        </section>
    </main>
</div>
```

- `.section` — max-width 900px, centered, full viewport height, `scroll-margin-top` for nav offset.
- `.section.wide` — max-width 1100px.
- Section `id` enables scroll-based nav highlighting via `initTopNav()`.

### With Sidebar

For slides with side notes:

```html
<div class="page">
    <aside class="sidebar">
        <div class="sidebar-note" data-for="section-id">
            Note content shown when section is in view.
        </div>
    </aside>
    <main class="main with-sidebar">
        <section class="section" id="section-id">...</section>
    </main>
</div>
```

## JavaScript Utilities (`shared.js`)

| Function | Purpose |
|---|---|
| `initSidebarNotes()` | Shows/hides sidebar notes based on which section is in view. |
| `initTopNav()` | Highlights active nav link based on scroll position. |
| `initKeyNav(prev, next)` | Arrow-key navigation. `null` for first/last. Auto-injected by build. |
| `initScrollFade()` | Adds `.visible` class to `.fade-on-scroll` elements when scrolled into view. |
| `typewriter(el, text, speed)` | Types text character-by-character into `el`. Returns a promise. |
| `staggerAppear(container, selector, delay)` | Staggers opacity/transform animation on matched elements. |
| `wait(ms)` | Promise-based delay. |

All init functions are auto-called on `DOMContentLoaded`.

## Animation Recipes

### Fade on Scroll

```html
<div class="fade-on-scroll">Content that fades in</div>
```

Managed by `initScrollFade()`. No extra JS needed.

### Typewriter

```js
const el = document.getElementById('target');
typewriter(el, 'Text to type', 50);
```

### Stagger

```js
staggerAppear('.container', '.item', 100);
```

### CSS Keyframes (define in per-slide `<style>`)

```css
@keyframes fadeUp { from { opacity: 0; transform: translateY(20px); } to { opacity: 1; transform: translateY(0); } }
@keyframes float { 0%, 100% { transform: translateY(0); } 50% { transform: translateY(-10px); } }
@keyframes shimmer { 0% { background-position: -200% center; } 100% { background-position: 200% center; } }
```

Apply with:

```css
.element { animation: fadeUp 0.6s ease forwards; }
.element:nth-child(2) { animation-delay: 0.1s; }
```

## Interactive Patterns

Use vanilla JS in an IIFE within an inline `<script>` block:

```html
<script>
(function() {
    const buttons = document.querySelectorAll('.mode-toggle button');
    const panels = document.querySelectorAll('.mode-panel');
    buttons.forEach(btn => {
        btn.addEventListener('click', () => {
            buttons.forEach(b => b.classList.remove('active'));
            btn.classList.add('active');
            panels.forEach(p => p.style.display = 'none');
            document.getElementById(btn.dataset.target).style.display = 'block';
        });
    });
})();
</script>
```

Place inline scripts between `<!-- partial:scripts -->` and `<!-- partial:keynav -->`.

## Responsive Design

- **1024px breakpoint**: sidebar shrinks to 260px.
- **768px breakpoint**: sidebar hidden, main content full-width.
- Always include `<meta name="viewport">` (handled by head partial).
- Use `max-width` on `.section` for content width (900px default, 1100px `.wide`).

## Checklist: Adding a New Slide

1. Create `NN-slug.html` in your deck directory.
2. Use partial markers for head, topnav, scripts, and keynav.
3. Add the slide entry to `deck.json` in the correct position.
4. Run `npm run build` — topnav and keynav are auto-generated.
5. Verify in browser.

## Checklist: Removing a Slide

1. Delete the HTML file.
2. Remove the entry from `deck.json`.
3. Run `npm run build` — topnav and keynav auto-update.

## Checklist: Creating a New Deck

1. Create `decks/[deck-name]/` directory.
2. Create `deck.json` with title and slides array.
3. Create `index.html` redirect.
4. Create slide files with partial markers.
5. Run `npm run build`.
6. Open `dist/[deck-name]/index.html` to verify.
